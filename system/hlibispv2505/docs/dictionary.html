Dictionary¶

This section contains the abbreviations used for the software and hardware components as well as the common acronyms used in the documentation.

Acronyms¶

C

CI:
Capture Interface (DDK library).
CSIM:

C Simulator.

Bit accurate software version of the hardware used for verification and early software development.

D

DDK:

Driver Development Kit.

By extension used as the software package delivered.

H

HAL:

Hardware Abstraction Layer.

In Android layer to be implemented when adding devices support (e.g. Camera HAL).

HW:
Hardware

I

I2C:

Inter-Integrated Circuit.

A bus standard, pronounced I-square-C or I-two-C.

ION:
Shared memory allocator. Used in Android.
ISP:
Image Signal Processor.
ISPC:
ISP Control (DDK library).

M

MIPI:

Mobile Industry Processor Interface.

In this document this usually refers to a CSI-2 specification. See TRM for more details.

N

NUMA:

Non-Uniform Memory Access.

A computer memory design, where the memory access time depends on the memory location relative to the processor.

P

PDP:

Pixel Display Pipeline.

Imagination IP to display images on screen.
PHY:
Physical layer IP for an interconnection interface (e.g image input/output).

S

SCB:
Serial Communication Block. Imagination’s IP to handle I2C.
SW:
Software

T

TAL:
DDK library to access HW registers.
TRM:

Technical Reference Manual.

By extension the delivered V2500 hardware document (not provided as part of software package).

V2500 modules¶

This section provides the list of HW and SW modules abbreviations as used in the documentation and code.

HW Pipeline modules (adapted from HW TRM diagram).

In Pipeline order:

Modules¶

GAS:

Gasket.

[Data from there on is Bayer RGGB].

IIF:
Imager Interface (ISPC: Imager Interface (IIF)).
BLC:
Black Level Correction (ISPC: Black Level Correction (BLC)).
RLT:
Raw Look Up Table (HW v2 only, register space only reserved in HW v1). (ISPC: Raw Look-up Table (RLT)).
LSH:
Lens Shading, also contains register for SW module: WBC (ISPC: Lens Shading (LSH)).
WBC:
SW only White Balance Correction, part of LSH module in HW (ISPC: White Balance Correction (WBC)).
FLS:
Filter Line Store (no registers - linestores are configured at top-level by SW).
DNS:
Primary Denoiser (ISPC: Denoiser (DNS/SHA_DN)).
DPF:
Defective Pixel Fixing, also considered as output and input ( Defective Pixels Fixing (DPF)).
LCA:
Lateral Chromatic Aberration (ISPC: Lateral Chromatic Aberration (LCA)).
DMO:

Demosaicer (no registers - DMO configured using IIF information)

[Data is converted from Bayer to RGB here].

CCM:
Colour Correction Matrix (ISPC: Colour Correction Matrix (CCM)).
MGM:
Main Gamut Mapper (ISPC: Main Gamut Mapper (MGM)).
GMA:
Gamma Look Up Table - only to enable or not the usage of values stored into the GMA_LUT register bank.
R2Y:

RGB to YUV converter (ISPC: RGB to YUV (R2Y)).

[Data is converted from RGB to YUV 444 here].

MIE:
Main Image Enhancer, SW uses MIE for the Memory Colour part of the HW block - also contains VIB module (ISPC: Memory Image Enhancer (MIE)).
VIB:
SW only Vibrancy, part of MIE module in HW (ISPC: Vibrancy (VIB)).
HSS:

Horizontal sub-sampler (no registers).

[Data is converted from YUV 444 to YUV 422 here].

TNM:
Tone Mapper (ISPC: Tone Mapper (TNM)).
FLD:
Flicker Detection, also considered an output (Flicker Detection (FLD)).
TLS:
Tone Mapper Line Store (no registers - linestores are configured at top-level by SW).
SHA:
Sharpening (ISPC: Sharpening (SHA) and Denoiser (DNS/SHA_DN)).
SLS:
Scaler Line Stores (no registers - linestores are configured at top-level by SW).
ESC:

Encoder Scaler (ISPC: Encoder Pipeline Scaler (ESC)).

[Data is converted from YUV 422 to YUV 420 optionally here (or in VSS)].

VSS:

Vertical Sub-sampler (no registers - information is from ESC).

[Data is converted from YUV 422 to YUV 420 optionally here (or in ESC)].

DSC:
Display Scaler (ISPC: Display Pipeline Scaler (DSC)).
HUS:

Horizontal Up-sampler (no registers).

[Data is converted from YUV 422 to YUV 444 here].

Y2R:

YUV to RGB converter (ISPC: YUV to RGB (Y2R)).

[Data is converted from YUV to RGB here].

DGM:
Display Gamut Mapper (ISPC: Display Gamut Mapper (DGM)).

Outputs¶

All high level setups are in the ISPC Output formats (OUT) section.

DEX1:
(not in diagram) - Image data Extraction just before BLC module (Bayer).
DEX2:
(not in diagram) - Image data Extraction just before FLS module (Bayer).
Encode:
(Luma and Chroma) - YUV output at the end pipeline.
Display:
RGB output at the end pipeline.
RAW 2D Extraction:
Bayer extraction just before DMO module (Bayer in TIFF format).
HDR Extraction:
RGB extraction just before HDF module.

Statistics outputs¶

EXS:
Exposure Clipping Statistics (ISPC: Exposure Statistics (EXS)).
FOS:
Focus statistics (ISPC: Focus Statistics (FOS)).
DPF:
Defective Pixel Fixing, optional output of fixed map input ( Defective Pixels Fixing (DPF)).
AWS:
Auto White Balance Statistics providing data useful for AWB Planckian Locus algorithm. (ISPC: Auto White Balance Statistics (AWS)).
WBS:
White Balance statistics (ISPC: White Balance Statistics (WBS)).
HIS:
Histogram (ISPC: Luma Histogram Statistics (HIS)).
FLD:
Flicker Detection (ISPC: Flicker Detection (FLD)).
ENS:
Encoder Statistics (ISPC: Encoder Statistics (ENS)).

Image Inputs¶

DG, ExtDG:
External Data Generator, piece of HW connected to the gasket on Imagination’s FPGA system and CSIM (no available on production chips).
IIFDG or intDG:
Internal Data Generator, similar to DG but available on production chips (Parallel Gasket only).
HDF:
HDR insertion, insert a high dynamic range image (designed to be input of the merging of several images extracted using HDR Extraction).
