This section contains the frequently asked questions and has links to other parts of this documentation that solves them.

How to read/write hardware registers for debug purposes:

This is covered by the low level driver (CI) documentation: Register access for debugging purposes.

When using the high level library (ISPC) one can use or modify the PrintGasketInfo() and PrintRTMInfo() in info_helper.cpp.

Debugging an MMU page fault:
See CI Appendix: Debugging a Page Fault.
Debugging a fail to aqcuire buffer (missed frame):
See CI Appendix: Debugging a Failed to acquire frame.
Allocating buffers with specific size requirements (stride, memory layout):

See CI documentation: CI: Output sizes and CI: Output layout.

See ISPC documentation ISPC: Output sizes and ISPC: Output layout.

Porting driver to a new platform:

This is covered by the platform integration guide and is usually done in two steps:

  • Integration of the device access and discovery (Device Access)
  • Integration of the memory access (Memory management)
Implementing a new sensor driver:
This is covered by the platform integration guide (Sensor driver).
