Detailed Description

The pipeline configuration.

Field Documentation

IMG_BOOL8 CI_PIPELINE_CONFIG::bSupportTiling

Use to enable allocation/importation of tiled output buffers.

IMG_BOOL8 CI_PIPELINE_CONFIG::bUseEncOutLine

To try to use the Encoder output line in HW with this context (only with supported encoder HW)

Note
Written to FELIX_CORE:ENC_OUT_CTRL when starting the capture but has to be specified before registration to the kernel-side
Warning
Will fail to start if another context is using it

See CI_PIPELINE::ui8EncOutPulse

enum CI_INOUT_POINTS CI_PIPELINE_CONFIG::eDataExtraction

If CI_INOUT_NONE then Data Extraction is done into the display buffer.

Encoder output format.

PIXELTYPE CI_PIPELINE_CONFIG::eHDRInsType

HDR Insertion format - HW supports only BGR_161616_64.

IMG_UINT32 CI_PIPELINE_CONFIG::eOutputConfig

PIXELTYPE CI_PIPELINE_CONFIG::eRaw2DExtraction

Raw 2D Extraction output format - HW supports only TIFF format here.

IMG_UINT16 CI_PIPELINE_CONFIG::ui16MaxDispOutHeight

Used to display output buffer allocation - further update of DSC output size cannot be bigger than that.

IMG_UINT16 CI_PIPELINE_CONFIG::ui16MaxDispOutWidth

Used to display output buffer allocation - further update of DSC output size cannot be bigger than that.

IMG_UINT16 CI_PIPELINE_CONFIG::ui16MaxEncOutHeight

Used to encoder output buffer allocation (in pixels) - further update of ESC output size cannot be bigger than that.

IMG_UINT16 CI_PIPELINE_CONFIG::ui16MaxEncOutWidth

Used to encoder output buffer allocation (in pixels) - further update of ESC output size cannot be bigger than that.

IMG_UINT16 CI_PIPELINE_CONFIG::ui16SysBlackLevel

Defines the black point value of pixels for modules that make use of unsigned offset pixel values.

Black level value is set according to bit depth of Felix pipeline irrespective of source bit depth and module to which it is applied.

Note
Register: SYS_BLACK_LEVEL:SYS_BLACK_LEVEL
IMG_UINT32 CI_PIPELINE_CONFIG::ui32DPFWriteMapSize

DPF output map size in bytes (if 0 no DPF write map allocated)

Note
Written to load Structure DPF_MODIFICATIONS_MAP_SIZE_CACHE:DPF_WRITE_MAP_SIZE when triggering shots
Warning
Once registers to the kernel this option cannot be changed
IMG_UINT8 CI_PIPELINE_CONFIG::ui8Context

HW context to use.

IMG_UINT8 CI_PIPELINE_CONFIG::ui8EncOutPulse
Note
written to FELIX_CORE:ENC_OUT_CTRL if using the Encoder output line but has to be specified before registration to the kernel-side

See CI_PIPELINE::bUseEncOutLine

IMG_UINT8 CI_PIPELINE_CONFIG::ui8PrivateValue

Written to registers - this value is copied to the shot's equivalent when added to the pending list.

Note
E.g. use this value to monitor which capture's setup was used for a shot.

See CI_SHOT::ui8PrivateValue
