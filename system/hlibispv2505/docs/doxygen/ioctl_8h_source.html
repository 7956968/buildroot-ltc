15 /* ioctl command encoding: 32 bits total, command in lower 16 bits,
16  * size of the parameter structure in the lower 14 bits of the
17  * upper 16 bits.
18  * Encoding the size of the parameter structure in the ioctl request
19  * is useful for catching programs compiled with old versions
20  * and to avoid overwriting user space outside the user buffer area.
21  * The highest 2 bits are reserved for indicating the ``access mode''.
22  * NOTE: This limits the max parameter size to 16kB -1 !
23  */
