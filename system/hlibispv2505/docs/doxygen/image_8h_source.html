248 /*IMPORTANT
249 FLX format supports multiple segments with multiple frames per segment. All frames in a segment share their own meta data.
250 Therefore, each segment is like a separate file with its own format. At most one frame is held in memory at any time and
251 normally the 'current' segment (pCurrSeg) points to the segment which holds the frame's meta data.
252 
253 -- LOADING --
254 When a file is loaded, LoadFileHeader() scans the file and loads information about all segments into pSegInfo list.
255 ApplyMetaForFrame() is used to set the 'current' segment as appropriate for the particular frame and a subsequent
256 LoadFileData() call will load frame into memory (it relies on the correct meta data, i.e. current segment, to be selected).
257 
258 -- SAVING --
259 If CImageFlx object has segment information available, it can be used to save the file, i.e. ApplyMetaForFrame() can be
260 called to select an appropriate segment before calling SaveFileHeader() and SaveFileData(). In this case FLXSAVEFORMAT.pMD
261 can be NULL and saving will pick up meta data from the current segment.
262 Alternatively, FLXSAVEFORMAT.pMD can point to separately created meta data, or meta data of another CImageFlx object.
263 The FLXSAVEFORMAT.pImData pointer can point to another image from which pixel data is being saved. Useful for conversions.
264 */
