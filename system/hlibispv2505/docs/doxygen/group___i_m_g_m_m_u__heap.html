IMGMMU_HeapAlloc* IMGMMU_HeapAllocate ( IMGMMU_Heap *  pHeap,
IMG_SIZE  uiSize,
IMG_RESULT *  pResult 
)

Allocate from a heap.

Warning
Heap do not relate to each other, therefore one must insure that they should not overlap if they should not.
Parameters
pHeapmust not be NULL
uiSizeallocation size in bytes
pResultmust be non-NULL - used to give details about error
Returns
pointer to the new HeapAlloc object and pResult is IMG_SUCCESS
NULL and the value of pResult can be:
  • IMG_ERROR_INVALID_PARAMETERS if the give size is not a multiple of pHeap->uiAllocAtom
  • IMG_ERROR_MALLOC_FAILED if the internal structure allocation failed
  • IMG_ERROR_NOT_SUPPORTED if the internal device memory allocator did not find a suitable virtual address
IMGMMU_Heap* IMGMMU_HeapCreate ( IMG_UINTPTR  uiVirtAddrStart,
IMG_SIZE  uiAllocAtom,
IMG_SIZE  uiSize,
IMG_RESULT *  pResult 
)

Create a Heap.

Parameters
uiVirtAddrStartstart of the heap - must be a multiple of uiAllocAtom
uiAllocAtomthe minimum possible allocation on the heap in bytes
  • usually related to the system page size
uiSizetotal size of the heap in bytes
pResultmust be non-NULL - used to give detail about error
Returns
pointer to the new Heap object and pResult is IMG_SUCCESS
NULL and the value of pResult can be:
  • IMG_ERROR_MALLOC_FAILED if internal allocation failed
