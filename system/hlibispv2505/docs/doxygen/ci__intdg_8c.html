IMG_RESULT CI_DatagenInsertFrame ( CI_DG_FRAME *  pFrame)

Insert a frame for capture.

To cancel a frame reservation without triggering use CI_DatagenReleaseFrame()

If inserted correctly the completion should be waited on using CI_DatagenWaitProcessedFrame() at a later point.

Returns
IMG_SUCCESS
IMG_ERROR_NOT_SUPPORTED if blanking is too small or the stride is not aligned to the system alignment
IMG_ERROR_UNEXPECTED_STATE if the given frame format is not supported by the data-generator
IMG_ERROR_INVALID_PARAMETERS if the frame is not found or the given stride*height is bigger than the allocated size

Returns

IMG_RESULT CI_DatagenStart ( CI_DATAGEN *  pDatagen)

Acquires needed HW for the capture.

Maps elements to device MMU in kernel (all frames must be allocated before that)

Returns
IMG_SUCCESS (if already started returns directly)
IMG_ERROR_INVALID_PARAMETERS if pDatagen is NULL
IMG_ERROR_FATAL if datagen object was not found in the kernel-side
IMG_ERROR_COULD_NOT_OBTAIN_RESOURCE if chosen datagen resource is not available
IMG_RESULT CI_DatagenStop ( CI_DATAGEN *  pDatagen)

Releases acquired HW for the capture.

unmaps elements from device MMU in kernel (frames can be destroyed after that)

Returns
IMG_SUCCESS (if already stopped returns directly)
IMG_ERROR_INVALID_PARAMETERS if pDatagen is NULL
IMG_ERROR_FATAL if datagen object was not found in the kernel-side

IMG_BOOL8  bBlocking 
)
