Detailed Description

Converts and submit images to the data generator HW.

Usage of a camera object is: creation (DG_CameraCreate), then the user can configure.

Buffer can then be added (DG_CameraAllocateFrame). When the capture started (DG_CameraStart) the configuration is updated.

Available buffers are accessed with DG_CameraGetAvailableFrame and can be populated using the CI_CONVERTER functions. Once ready the frame is pushed with DG_CameraInsertFrame or the capture is cancelled with DG_CameraReleaseFrame.

Once finished the capture should be stopped (DG_CameraStop) and the camera can be destroyed (DG_CameraDestroy).

Insert a frame for the capture.

The frame should have been acquired with DG_CameraGetAvailableFrame() or an equivalent function and convertion should have been made to the data if needed.

IMG_RESULT DG_CameraStart ( DG_CAMERA *  pCamera)

Start the capture - acquire the needed HW for captures.

Registers the needed config in kernel-side and allow frames to be triggered.

Kernel-side can refuse a camera to be started if the HW is not available.

Warning
The CI gasket has to be configured and acquired from another function!
Returns
IMG_SUCCESS
IMG_ERROR_INVALID_PARAMETERS if pCamera is NULL
IMG_ERROR_NOT_INITIALISED if the Camera is not registered to kernel-side (unlikely)
IMG_ERROR_NOT_SUPPORTED if the Camera is already started
IMG_ERROR_FATAL if kernel-side refused to start this camera
