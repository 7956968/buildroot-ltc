IMG_RESULT HW_CI_PipelineRegStart ( KRN_CI_PIPELINE *  pCapture)

Write the registers when starting the capture.

The HW is considered as "fail to start" if:

  • the capture mode is not disbaled, or
  • the linked list emptiness is 0
Returns
IMG_SUCCESS
IMG_ERROR_NOT_SUPPORTED if the capture is not marked as started in SW (KRN_CI_PIPELINE::bStarted)
IMG_ERROR_FATAL if the bayer format is unknown, or the HW failed to start
void HW_CI_PipelineRegStop ( KRN_CI_PIPELINE *  pCapture,
IMG_BOOL8  bPoll 
)

Write the registers to stop the capture.

Done in that order:

  • write interrupt enabled to 0 to stop receiving interrupts for that context
  • make the HW in disable mode
  • if bPoll waits for context to become idle
  • clear the linked list

unsigned int  eConfigOutput,
IMG_UINT32  ui32LoadConfigReg

Configures the HW Linked List from the Shot structure and its associated pipeline.

This function updates the linked list enable flags according to the present buffers and enable flags.

Use HW_CI_ShotUpdateAddresses() to update the output addresses.

Parameters
[in,out]pBuffer
[in]eConfigOutputsee CI_PIPELINE_CONFIG::eOutputConfig
[in]ui32LoadConfigRegas a combinaison of CI_MODULE_UPDATE
Warning
Image buffers should already be allocated.
Note
call this if updates in the pipeline are needed (e.g. new statistics output)

@ could compute the correct flags for loading only the needed elements
